## 1. 게임 개요

체스의 이동 규칙을 기반으로 하되, 매 턴 마나를 적립하여 새로운 체스말을 왕 주변에 소환할 수 있는 전략 게임.
초반엔 소수의 폰으로 버티다가, 마나가 쌓이면 강력한 말을 소환해 전세를 역전시키는 흐름이 핵심 재미다.

### 핵심 재미 요소

- **소환 타이밍의 긴장감**: 마나를 아껴 퀸을 소환할 것인가, 지금 당장 폰을 소환해 빈틈을 막을 것인가
- **왕 중심 방어선**: 소환은 왕 인접에서만 가능하므로, 왕을 지키면서 공격 거점도 구축해야 함
- **제한된 보드**: 5×5라는 좁은 공간에서 순식간에 판세가 뒤집힘
- **60초 턴 제한**: 시간 압박이 더해져 직관적 판단이 중요

### 플랫폼

| 플랫폼 | 상태 |

## 2. 게임 규칙

- **보드:** 5×5 격자
- **플레이어:** 플레이어 vs AI
- **승리 조건:** 상대 왕 포획 또는 체크메이트

### 시작 배치

```
행 0: [  .  ][  .  ][AI K][  .  ][  .  ]   AI 왕 (중앙 고정)
행 1: [AI P][AI P][  .  ][AI P][AI P]       AI 폰 4개
행 2: [  .  ][  .  ][  .  ][  .  ][  .  ]  빈 칸
행 3: [  .  ][  .  ][  .  ][  .  ][  .  ]  플레이어 배치 구역
행 4: [  .  ][  .  ][PL K][  .  ][  .  ]   플레이어 왕 (중앙 고정)
```

- **쉬움/보통**: 배치 페이즈 없이 자동 랜덤 배치 후 즉시 게임 시작
- **어려움**: 배치 페이즈에서 하단 2행(행 3~4, 왕 칸 제외)에 폰 4개를 자유 배치

### 턴 순서

1. 턴 시작 — 마나 +2 (최대 10)
2. 이동 또는 소환 선택 (턴당 하나만)
3. 행동 완료 또는 60초 초과 시 턴 종료
4. 상대 턴으로 전환 (플레이어 ↔ AI 교대)

- 수동 턴 종료: "턴 종료" 버튼
- 기권: "기권" 버튼 클릭 시 패배 처리

### 이동 규칙

| 말 | 이동 |
|----|------|
| 폰 (P) | 전방 1칸, 대각선 포획 |
| 나이트 (N) | L자 점프 |
| 비숍 (B) | 대각선 슬라이드 |
| 룩 (R) | 직선 슬라이드 |
| 퀸 (Q) | 비숍 + 룩 |
| 왕 (K) | 인접 1칸 (공격받는 칸으로는 이동 불가) |

- 캐스링·앙파상 등 특수 규칙 없음
- 플레이어 폰은 위(행 감소), AI 폰은 아래(행 증가)

### 폰 승급

폰이 상대편 끝 행(플레이어: 행 0, AI: 행 4)에 도달하면 퀸으로 자동 승급.

### 체크 / 체크메이트

- 왕이 공격받으면 체크 표시 (빨간 링 + 깜박임 연출)
- 체크메이트 판정: 이동으로 해제 불가 + 소환으로도 해제 불가 → 패배
- 시간 초과 시 턴 종료 처리 (강제 기권 아님)

## 3. 마나 & 소환 시스템

| 항목 | 값 |
|------|----|
| 턴당 충전 | +2 |
| 최대 마나 | 10 |
| 시작 마나 | 0 |

### 소환 비용

| 말 | 기본 비용 | 같은 종류 N번째 소환 시 비용 |
|----|-----------|-------------------------------|
| 폰 (P) | 1 | 1 + (소환 횟수) |
| 나이트 (N) | 3 | 3 + (소환 횟수) |
| 비숍 (B) | 3 | 3 + (소환 횟수) |
| 룩 (R) | 5 | 5 + (소환 횟수) |
| 퀸 (Q) | 8 | 8 + (소환 횟수) |

- 같은 종류를 반복 소환할수록 비용이 1씩 증가 (소환 누적 인플레이션)
- UI 소환 버튼에 현재 실제 비용이 동적으로 표시됨

### 소환 규칙

- 소환 위치: 자신의 왕 기준 인접 8칸 중 빈 칸만 가능
- 소환하면 이동 불가, 이동하면 소환 불가 (턴당 1회)
- 소환 직후 해당 말은 그 턴에 이동 불가 (다음 턴부터 이동 가능)
- 왕은 소환 불가 (처음부터 배치)

### 소환 버튼 조작

- 소환 버튼 클릭 → 소환 대기 모드 진입 (소환 가능 칸 초록 하이라이트)
- 같은 버튼 재클릭 → 소환 취소 (WAITING 상태로 복귀)
- 다른 버튼 클릭 → 소환 대상 변경 (바로 전환)

## 4. UI / 화면 구성

### 씬 흐름

```
메인 메뉴 (난이도 선택)
    ↓
배치 페이즈 (어려움만 / 쉬움·보통은 자동 처리)
    ↓
게임 (보드 + HUD 오버레이)
    ↓
결과 화면 (승 / 패)
    ↓
다시하기 → 배치 페이즈  /  메인 메뉴 → 메인 메뉴
```

### 인게임 HUD (우측 패널)

| 위치 | 표시 내용 |
|------|-----------|
| 상단 | 현재 턴 표시 (내 턴 / AI 턴) |
| 타이머 | 60초 카운트다운 (10초 이하: 빨간색) |
| 마나 | 현재 마나 / 최대(10) |
| 행동 현황 | ○/✓ 이동, ○/✓ 소환 |
| 소환 버튼 | 5개 (마나 부족/이미 소환 시 비활성, 활성 시 주황 하이라이트) |
| 턴 종료 버튼 | 수동 턴 종료 |
| 기권 버튼 | 즉시 패배 처리 |

### 보드 시각 피드백

| 상황 | 표시 |
|------|------|
| 말 선택 | 노란색 하이라이트 |
| 이동 가능 칸 | 초록색 하이라이트 |
| 소환 가능 칸 | 초록색 하이라이트 |
| 체크 위협 왕 | 빨간 링 + 깜박임 애니메이션 |
| 활성 소환 버튼 | 주황색 강조 |

### 연출 / 애니메이션

| 이벤트 | 연출 |
|--------|------|
| 말 이동 | 부드러운 슬라이드 (트윈) |
| 포획 | 포획된 말 깜박임 후 제거 |
| 소환 | 황금빛 버스트 이펙트 |
| 체크 | 왕에 빨간 링, 2회 깜박임 |
| 턴 전환 | (추후 연출 추가 예정) |

### 전장의 안개 (Fog of War)

- 플레이어 말들의 실제 이동 경로 위 칸만 보임
- 나이트: L자 도착 칸 포함
- 슬라이드 말(비숍/룩/퀸): 경로 전체 칸
- **체크를 건 적 말은 안개와 무관하게 항상 표시**

## 5. AI 시스템

| 난이도 | 배치 | 이동 전략 | 소환 전략 |
|--------|------|-----------|-----------|
| 쉬움 | 자동 랜덤 | 랜덤 (자기 체크 회피 시도) | 랜덤 |
| 보통 | 자동 랜덤 | 포획 우선 (가치 높은 순, 자기 체크 제외), 이후 랜덤 | 마나 충족 시 랜덤 |
| 어려움 | 수동 배치 | Minimax + 알파베타 가지치기 (탐색 깊이 3) | 상황 평가 후 최적 소환 |

### 어려움 AI 평가 함수

```
점수 = Σ(자신의 말 가치) - Σ(상대 말 가치)
     + 중앙(3×3) 장악 보너스 (말당 +0.1)
     + 상대 체크 시 +2
     - 자신 체크 시 -2
     + 상대 체크메이트 시 +10000
```

## 6. 밸런스

### 구조적 긴장감

- **폰 중심 초반**: 마나 0에서 시작하므로 초반 2~3턴은 폰만으로 싸워야 함
- **왕 노출 리스크**: 소환하려면 왕 주변 칸이 필요하므로, 왕을 너무 가두면 소환 불가
- **소환 인플레이션**: 같은 종류 반복 소환 시 비용 +1 증가 → 단일 말 물량 전략 억제
- **좁은 보드 후반 강세**: 5×5에서 룩 1개만 있어도 열 전체를 장악

### 향후 밸런스 조정 포인트

- 마나 충전량 조정 (현재 2/턴 → 난이도별 차등 가능)
- 최대 마나 상한 조정 (현재 10)
- 소환 비용 개별 조정
- 소환 인플레이션 폭 조정 (+1 → 다른 값)
