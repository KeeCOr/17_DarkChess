## 1. 게임 개요

Chess Summon은 5x5 체스판에서 말을 지키며 마나로 새로운 말을 소환하는 전술 게임이다. 플레이어는 제한된 시야와 30초 턴 안에서 이동 1번, 소환 1번을 선택하고, 소환 위치를 확보해 전세를 뒤집어야 한다.

### 핵심 재미

- 마나를 모아 어떤 말을 언제 소환할지 판단하는 선택
- 제한된 보드에서 매 턴 거점을 확보하는 압박
- 안개가 걷히는 시야 정보를 추론하는 전술성
- 30초 제한으로 생기는 빠른 의사결정

---

## 2. 기본 규칙

- 보드: 5x5
- 모드: 싱글 플레이어 vs AI, 온라인 멀티 플레이 로비
- 승리 조건: 상대 왕 포획 또는 체크메이트
- 턴 시작 시 마나 +2, 최대 10
- 한 턴에 이동 1번, 소환 1번 가능
- 30초가 지나면 자동으로 턴 종료

### 시작 배치

```text
0행 [ . ][ . ][AI 왕][ . ][ . ]
1행 [AI 병][AI 병][ . ][AI 병][AI 병]
2행 [ . ][ . ][ . ][ . ][ . ]
3행 [ . ][ . ][ . ][ . ][ . ]
4행 [ . ][ . ][PL 왕][ . ][ . ]
```

사자와 보통은 병사 4명을 자동 배치한다. 어려움은 하단 2개 행에 병사 4명을 직접 배치한다.

---

## 3. 마나와 소환

| 말 | 기본 비용 |
| --- | ---: |
| 병사 | 1 |
| 기사 | 3 |
| 주교 | 3 |
| 성채 | 5 |
| 여왕 | 8 |

같은 종류를 반복 소환할수록 비용은 2씩 증가한다. 소환은 아군 말 주변 8칸의 빈 칸에만 가능하며, 소환된 말은 해당 턴에 이동할 수 없다.

## 4. UI / UX 변경 이력

### v0.1.17 난이도 선택 클릭 영역 개선

- 싱글 플레이 선택 후 난이도 버튼의 실제 클릭 영역을 버튼과 설명 문구를 포함한 카드 전체로 확장했다.
- 난이도 설명 문구 주변을 눌러도 해당 난이도로 배치 화면에 진입하도록 해 버튼이 안 눌리는 것처럼 느껴지는 문제를 줄였다.
- 난이도 카드 클릭 영역을 검증하는 회귀 테스트를 추가했다.

### v0.1.16 어려움 배치 후 전투 시작 수정

- 어려움 난이도에서 폰 위치를 바꾼 뒤 `전투 시작`을 누르면 현재 배치값으로 즉시 게임을 시작하게 했다.
- 전투 시작을 타이머 예약에 의존하지 않게 해 다시하기 이후 배치 화면에서 시작이 끊기거나 늦게 반응하는 문제를 막았다.
- 폰이 4개 배치되지 않은 상태에서는 내부 시작 함수가 호출되어도 게임을 시작하지 않도록 방어했다.

### v0.1.15 다시하기 멈춤 수정

- 결과 화면에서 `다시하기`를 누를 때 이전 게임의 `UI`, `Tutorial`, `Game` 씬을 먼저 정리한 뒤 배치 화면으로 이동하게 했다.
- 이전 게임의 보조 UI나 입력 리스너가 새 게임 위에 남아 멈춘 것처럼 보이는 문제를 막았다.
- 결과 화면에 난이도 정보가 없더라도 기본값을 사자로 보정해 재시작 흐름이 끊기지 않게 했다.

### v0.1.14 온라인 멀티 플레이 로비 및 계정 랭크

- 메인 메뉴에서 난이도 선택 전에 `싱글 플레이`와 `멀티 플레이`를 먼저 고르게 했다.
- 싱글 플레이는 기존처럼 난이도 선택으로 이어진다.
- 멀티 플레이는 닉네임을 입력하고 로컬 WebSocket 서버에 접속하는 온라인 로비로 이동한다.
- 계정별 랭크 포인트는 `data/rank-points.json`에 저장되며, 신규 계정은 1000점으로 시작한다.
- 빠른 매칭 버튼으로 대기열에 들어가고, 두 명이 접속하면 서로 매칭 완료 상태를 확인할 수 있다.
- 서버는 `npm run online:server`로 실행한다. 이번 버전은 계정/랭크/매칭 기반이며, 실제 대국 동기화는 다음 단계 확장 항목이다.

### v0.1.13 튜토리얼 턴 종료 및 제한시간 조정

- 일반 턴 제한시간을 60초에서 30초로 줄였다.
- 튜토리얼 중에는 타이머가 0이 되어도 자동으로 턴이 넘어가지 않게 했다.
- 이동과 소환을 모두 마친 뒤에도 턴 종료 단계에서는 반드시 `턴 종료` 버튼을 눌러야 다음 단계로 넘어가도록 했다.

### v0.1.12 튜토리얼 입력 제한 및 하이라이트 개선

- 튜토리얼에서 눌러야 하는 영역을 밝은 하이라이트로 표시했다.
- 눌러야 하는 영역 외에는 클릭 차단막으로 덮어 다른 입력이 동작하지 않게 했다.
- 안내만 보여주는 단계는 전체 화면을 차단하고 확인 버튼만 동작하게 했다.

### v0.1.11 반복 소환 마나 증가량 조정

- 같은 종류의 말을 반복 소환할 때 증가하는 마나 비용을 `+1`에서 `+2`로 변경했다.
- 실제 소환 판정과 오른쪽 소환 카드의 비용 표시가 같은 증가량 상수를 사용한다.

### v0.1.10 우클릭 이동 선택 취소

- 말을 선택해 이동 가능 칸이 표시된 상태에서 마우스 오른쪽 버튼을 누르면 이동 선택이 취소된다.
- 선택 취소 시 이동 하이라이트를 지우고 체크 위험 표시를 복구한다.

### v0.1.9 다시하기 및 튜토리얼 흐름 개선

- 결과 화면의 다시하기가 튜토리얼 질문 없이 바로 게임으로 진입하게 했다.
- 튜토리얼 오버레이가 실제 눌러야 하는 영역을 가리지 않도록 정리했다.
- 누락되었던 `한 턴에 이동 1번, 소환 1번 가능` 안내 단계를 복구했다.

### v0.1.7 선택 상태 턴 종료 개선

- 말을 선택한 상태에서도 `턴 종료` 버튼을 누르면 즉시 턴을 종료할 수 있게 했다.

### v0.1.6 HUD 겹침 및 소환 마나 위치 개선

- 보유 마나 표시를 소환 카드 섹션 내부로 이동해 소환 비용과 같은 맥락으로 보이게 했다.
- 오른쪽 HUD의 좌표와 간격을 상수화해 텍스트 겹침을 줄였다.

### v0.1.5 마나 비용 표기 개선

- 비용 텍스트를 제거하고 마나 아이콘과 숫자로 비용을 표시했다.
- 보유 마나와 비용 표기의 시각 언어를 통일했다.

### v0.1.4 말 가시성 및 HUD 겹침 수정

- 말 이미지를 256x256 정규화 에셋으로 재생성했다.
- 보드 위에서 말이 더 선명하고 크게 보이도록 렌더 크기를 조정했다.
- HUD 패널, 소환 카드, 하단 버튼 위치를 정리했다.

### v0.1.3 UI 겹침/글자 잘림/이미지 선명도 수정

- 오른쪽 HUD 문구를 공통 문구로 정리하고 패널 안에서 잘리지 않게 했다.
- 말 렌더 크기를 보드 칸 안에 들어오도록 제한했다.
- 깨진 공개 말 이미지를 원본 캐릭터 에셋으로 교체했다.

### v0.1.2 핵심 규칙/시야 학습 UX 개선

- 게임 화면을 보드 중심 + 오른쪽 운영 패널 구조로 다시 배치했다.
- 오른쪽 패널에 이동 1번, 소환 1번 행동 여부를 크게 표시했다.
- 보드 아래 상황 힌트가 현재 선택 상태에 따라 바뀌도록 했다.
- `?` 안내 버튼에서 소환 조건, 시야 조건, 턴 행동 규칙을 확인할 수 있게 했다.

### v0.1.1 다크 판타지 UX 개선

- 메뉴, 보드, HUD, 튜토리얼, 모달, 결과 화면의 시각 스타일을 통일했다.
- 보드 프레임, 안개 표현, 캐릭터 스프라이트 크기를 조정했다.
- 깨진 표기 문자열을 정상 문구로 복구했다.

### 화면 흐름

```text
메인 메뉴
  -> 싱글 플레이 / 멀티 플레이 선택
  -> 싱글: 난이도 선택
  -> 멀티: 닉네임 입력 및 매칭 로비
  -> 배치 또는 자동 배치
  -> 튜토리얼 여부 선택
  -> 게임 화면
  -> 결과 화면
```

## 5. 온라인 멀티 플레이

### 서버 실행

```bash
npm run online:server
```

### 랭크 저장

- 저장 위치: `data/rank-points.json`
- 신규 계정 기본 랭크 포인트: 1000
- 서버의 `recordResult` 함수는 승자 +15점, 패자 -15점으로 랭크를 갱신한다.
- 현재 클라이언트는 로비 접속, 계정 조회, 빠른 매칭까지 지원한다.
