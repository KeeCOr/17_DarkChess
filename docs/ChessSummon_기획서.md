## 1. 게임 개요

Chess Summon은 5x5 체스판에서 왕을 지키며 마나로 새로운 말을 소환하는 전술 게임이다. 플레이어는 제한된 시야와 60초 턴 안에서 이동 또는 소환을 선택하고, 소환 위치를 확보해 전세를 뒤집어야 한다.

### 핵심 재미

- 마나를 모아 어떤 말을 언제 소환할지 판단하는 선택
- 제한된 보드에서 왕 주변 거점을 확보하는 압박
- 안개에 가려진 적 움직임을 추론하는 전술성
- 짧은 턴 제한으로 생기는 빠른 의사결정

---

## 2. 기본 규칙

- 보드: 5x5
- 대전: 플레이어 vs AI
- 승리 조건: 상대 왕 포획 또는 체크메이트
- 턴 시작 시 마나 +2, 최대 10
- 한 턴에 이동 1회, 소환 1회 가능
- 60초가 지나면 자동으로 턴 종료

### 시작 배치

```text
0행: [ . ][ . ][AI 왕][ . ][ . ]
1행: [AI 병][AI 병][ . ][AI 병][AI 병]
2행: [ . ][ . ][ . ][ . ][ . ]
3행: [ . ][ . ][ . ][ . ][ . ]
4행: [ . ][ . ][PL 왕][ . ][ . ]
```

쉬움과 보통은 병사 4명을 자동 배치한다. 어려움은 하단 2개 행에서 병사 4명을 직접 배치한다.

---

## 3. 마나와 소환

| 말 | 기본 비용 |
| --- | ---: |
| 병사 | 1 |
| 기사 | 3 |
| 주교 | 3 |
| 성채 | 5 |
| 여왕 | 8 |

같은 종류를 반복 소환할수록 비용이 1씩 증가한다. 소환은 아군 말 주변 8칸의 빈 칸에만 가능하며, 소환된 말은 해당 턴에 이동할 수 없다.

## 4. UI / UX

### v0.1.7 선택 상태 턴 종료 개선

- 말을 선택해 이동 가능 칸이 표시된 상태에서도 `턴 종료` 버튼을 누르면 즉시 턴을 종료할 수 있게 했다.
- 턴 종료 시 선택된 말, 이동 하이라이트, 소환 대기 상태를 정리해 다음 턴 UI에 이전 선택 상태가 남지 않게 했다.
- 수동 턴 종료 가능 상태를 대기, 말 선택, 소환 위치 선택 상태로 확장하고 이동 애니메이션 중에는 기존처럼 입력을 막는다.

### v0.1.6 HUD 겹침 및 소환 마나 위치 개선

- 보유 마나 표시를 HUD 상단에서 소환 카드 섹션 내부로 이동해 소환 비용과 같은 맥락으로 보이게 했다.
- 소환 카드의 오른쪽 상태 문구를 제거하고 버튼 색상, 투명도, 비용 아이콘 상태로 선택 가능 여부를 전달해 글자 겹침과 잘림을 줄였다.
- 소환 안내, 마나 배지, 소환 카드 목록 사이의 Y 좌표를 레이아웃 상수로 분리해 이후 UI가 서로 침범하지 않도록 했다.
- 오른쪽 HUD 상단은 턴/타이머/도움말 중심으로 정리하고, 소환 영역은 보유 마나와 카드 목록 중심으로 다시 배치했다.

### v0.1.5 마나 비용 표기 개선

- 상단 마나와 소환 비용에 같은 청록색 보석 아이콘을 적용했다.
- 소환 카드의 `비용` 텍스트를 제거하고 마나 아이콘 + 숫자로 표시해 비용이 마나임을 직관적으로 알 수 있게 했다.
- 마나 표기와 비용 표기의 시각 언어를 통일했다.

### v0.1.4 말 가시성 및 HUD 겹침 추가 수정

- 말 이미지의 투명 여백을 제거하고 256x256 정규화 에셋으로 재생성했다.
- 정규화 후 실제 캐릭터 본체가 이미지 높이 대부분을 차지해 보드 위에서 더 선명하고 크게 보인다.
- 보드 말 렌더 크기를 86px로 조정해 칸 안에서 충분히 보이면서도 과도하게 넘치지 않게 했다.
- HUD 패널, 소환 카드 시작 위치, 행 간격, 하단 버튼 위치를 상수화해 겹침이 재발하지 않도록 했다.
- 소환 카드 내부 아이콘, 이름, 비용, 상태 문구의 위치를 다시 조정했다.

### v0.1.3 UI 겹침/글자 잘림/이미지 선명도 수정

- 오른쪽 HUD의 긴 하드코딩 문구를 짧은 공통 문구로 정리해 패널 안에서 잘리지 않게 했다.
- 타이머는 숫자 중심으로 표시해 상단 정보 영역의 겹침을 줄였다.
- 소환 카드 내부 텍스트 크기와 위치를 조정해 이름, 비용, 상태 문구가 겹치지 않게 했다.
- 80px 보드 칸 안에 말이 들어오도록 렌더 크기를 `74px`로 제한했다.
- 기존 72x72 public 말 이미지를 1024x1024 원본 캐릭터 에셋으로 교체해 확대 깨짐을 제거했다.

### v0.1.2 턴 규칙/시야 학습 UX 개선

- 게임 화면을 보드 중심 + 오른쪽 턴 운영 패널 구조로 다시 배치했다.
- 오른쪽 패널에 `이동 1회`, `소환 1회` 행동 슬롯을 크게 표시해 한 턴에 가능한 행동을 즉시 알 수 있게 했다.
- 이동 또는 소환을 완료하면 해당 슬롯이 `완료` 상태로 바뀌고, 남은 행동이 보드 아래 힌트에 표시된다.
- 보드 아래 상황 힌트가 현재 상태에 따라 바뀐다.
  - 기본: 말을 선택하거나 소환 카드를 선택하라는 안내
  - 말 선택: 이동하면 새 시야가 열린다는 안내
  - 소환 선택: 초록 칸을 클릭하면 소환된다는 안내
  - 행동 완료: 남은 행동 또는 턴 종료 안내
- `?` 도움말 버튼을 추가해 소환 조건, 시야가 밝혀지는 조건, 한 턴 행동 규칙을 언제든 확인할 수 있다.
- 소환 카드는 말 이름, 아이콘, 비용, 마나 부족 상태를 함께 보여준다.

### v0.1.1 다크 판타지 UX 개선

- 전체 톤을 어두운 판타지 전술판 스타일로 통일했다.
- 메뉴 첫 화면에 소환 의식 느낌의 배경 레이어와 난이도 설명을 추가했다.
- 보드 프레임, 금색 라인, 부드러운 안개 타일을 적용해 검은 덩어리처럼 보이던 시야 제한을 완화했다.
- 캐릭터 스프라이트 크기와 그림자를 조정해 말의 실루엣이 더 잘 보이도록 했다.
- 오른쪽 HUD를 턴, 타이머, 마나, 행동 현황, 소환, 종료 순서로 정리했다.
- 소환 버튼은 말 이름, 아이콘, 현재 비용을 함께 보여주며 활성/비활성/선택 상태가 명확하다.
- 튜토리얼, 기권 확인, 결과 화면의 패널 스타일을 동일하게 맞췄다.
- 깨진 한글 문자열을 정상 문구로 복구했다.

### 화면 흐름

```text
메인 메뉴
  -> 배치 또는 자동 배치
  -> 쉬움: 튜토리얼 여부 선택
  -> 게임 화면
  -> 결과 화면
```
